Sesión 2: Control e interactividad con Scratch

Sonidos, sensores, condicionales y variables

Objetivos de Aprendizaje

  • Incorporar sonidos para mejorar la expresión y retroalimentación en los proyectos.
  • Utilizar sensores para detectar interacciones con el teclado, mouse, colores u objetos.
  • Aplicar condicionales si...entonces y si...entonces...si no para controlar decisiones según condiciones del entorno.
  • Crear y modificar variables para gestionar información como puntajes, tiempo o estados del juego.

Sonidos

Incorporación de sonidos a los proyectos y exploración de la pestaña de sonidos (Tutorial pág. 21).

Bloques de sonido:

  • tocar sonido [miau]: Reproducir sonido completo

  • tocar sonido [miau] hasta que termine: Esperar a que termine

  • detener todos los sonidos: Silenciar proyecto

  • cambiar volumen por [10]: Ajustar volumen

Ejemplo práctico:

Reproducir un maullido cuando el gato es presionado, o agregar sonidos de pasos al caminar.

Sensores

Los sensores permiten que los proyectos respondan a estímulos externos y condiciones del entorno, haciendo las animaciones y programas más interactivos y dinámicos.

Tipos de sensores comunes:

  • Sensor de contacto: Detecta si un objeto toca otro o el borde del escenario.

  • Sensor de posición del ratón: Permite detectar la posición del cursor para interacciones.

  • Sensor de distancia: Mide la distancia entre objetos o al cursor.

  • Sensor de color: Detecta si un objeto toca un color específico.

  • Sensor de sonido: Detecta niveles de sonido para activar acciones.

Condicionales y Variables

Condicionales

En Scratch, una condicional es una instrucción que permite tomar decisiones durante la ejecución del programa.

Bloques de Condiciones:

  • Bloque si... entonces: Este bloque ejecuta el código que tiene dentro solo si la condición que se le da es verdadera. Si la condición es falsa, el código se ignora.

  • Bloque si... entonces... si no: Este bloque ofrece dos caminos. Si la condición es verdadera, ejecuta el código en la sección si. Si la condición es falsa, ejecuta el código en la sección si no.

Ejemplo práctico:

Si el gato avanza nos dice "caminando", si el gato no avanza nos dice "quieto",

Variables

Una variable en Scratch es una caja que guarda un valor que puede cambiar durante el programa.

Tipos de variables:

  • Variables globales: Accesibles por todos los objetos

  • Variables locales: Solo para el objeto que las crea

Bloques de variables:

  • fijar [variable] a [valor]: Asignar valor inicial

  • cambiar [variable] por [cantidad]: Incrementar/decrementar

  • mostrar variable [variable]: Hacer visible en escenario

  • esconder variable [variable]: Ocultar del escenario

🧩Desafío: El Gatito Saltador de Energía

  • Cuando presiones la bandera verde, el gato debe aparecer en el centro del escenario con su energía inicial fijada en 10. Luego de mostrarse, debe dar un mensaje breve con instrucciones que diga: “Presiona ESPACIO para saltar - R para recargar”. A lo largo del juego, si se presiona la tecla ESPACIO y aún tiene energía, el gato debe simular un salto (subiendo y bajando rápidamente), sonar un "meow", disminuir su energía en 1 y decir “¡Salto!”. Si no tiene energía, dirá “¡Sin energía!”. Además, si se presiona la tecla R, el gato recargará 1 punto de energía y mostrará el mensaje “Recargando...”.
